# Installation Overview

Poweradmin can be installed on various operating systems and with different database backends. This section provides installation guides for different environments.

## Operating Systems

- [Debian](debian.md) - Installation guide for Debian
- [Ubuntu](ubuntu.md) - Installation guide for Ubuntu
- [CentOS/RHEL](centos.md) - Installation guide for CentOS, Rocky Linux, AlmaLinux, and Red Hat Enterprise Linux

## Installation Methods

- [Web Installer Wizard](wizard.md) - Recommended for first-time installs; an 8-step browser-based walkthrough
- [Manual Installation](manual.md) - Set everything up by hand without the wizard (suitable for automated deployments)
- [Composer Installation](composer.md) - Installing Poweradmin using Composer
- [Docker](docker.md) - Run Poweradmin in a container
- [Docker Secrets](docker-secrets.md) - Use Docker / Swarm secrets for credentials
- [Reverse Proxy](reverse-proxy.md) - Front Poweradmin with nginx, Traefik, or Caddy as a reverse proxy
- [Remote Setup Guide](remote-setup-guide.md) - Install Poweradmin and PowerDNS on separate hosts

## Database Setup

- [MySQL/MariaDB](/database/mysql-configuration.md) - Setting up with MySQL or MariaDB
- [PostgreSQL](/database/postgresql-configuration.md) - Setting up with PostgreSQL
- [SQLite](/database/sqlite.md) - Setting up with SQLite

Choose the installation method that best fits your environment and requirements.
